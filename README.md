# Python工程师笔试题

## 📋 考试说明

恭喜你通过了技术面试！本次将通过一个实际项目来全面评估你的技术能力。

### 时间安排
- **开发时间**：1天（建议8-12小时）
- **演示时间**：预定的二面时间

---

## 📦 提交内容

### ⏰ 提交截止时间
**二面开始前**请将以下材料发送至：**john.huang@deep-edge.ai**

### 必需提交
1. **源代码**：完整的、可运行的代码
2. **README.md**：项目说明、技术选型、安装步骤
3. **docs/DESIGN.md**：架构设计、设计决策说明
4. **演示材料**：15分钟演示的准备（PPT或录屏）

### 在文档中说明
- 你选择的技术栈及理由
- 架构设计思路
- 遇到的挑战和解决方案
- 如果有更多时间会怎么改进

---

## 🤖 AI使用规范

### ✅ 鼓励使用
- 代码框架生成
- 调试和错误排查
- 文档优化和格式调整
- 学习新技术API

### 📝 透明度要求
请在README中添加：
```markdown
## 🤖 AI使用说明

简要说明哪些部分使用了AI辅助，以及你对生成代码的理解程度。
```

### ⚠️ 注意事项
- 确保理解每一行提交的代码
- 准备在演示时讲解任何代码段
- 不要提交自己无法解释的代码

## 📊 评分标准

| 评分项 | 权重 | 说明 |
|----------|------|------|
| **功能完整性** | 30% | 核心功能实现、边界处理、错误处理 |
| **架构设计** | 25% | 技术选型合理性、系统设计思路 |
| **代码质量** | 20% | 可读性、规范性、测试 |
| **文档质量** | 15% | 设计文档、技术说明完整性 |
| **技术亮点** | 10% | 创新点、优化、工程实践 |

## 演示环节（15分钟）

### 建议结构
- **2分钟**：项目介绍和架构设计
- **7分钟**：功能演示
- **6分钟**：技术实现讲解
- **15分后**：问答

### 准备建议
- 准备测试文档和问题
- 提前测试演示环境
- 思考可能的追问
