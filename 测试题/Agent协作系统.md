# 方向A：AI/LLM应用

## 📌 难度2：多Agent协作工作流系统

> **预计时间**：8-12小时

---

## 项目背景

单一Agent往往难以处理复杂任务。设计并实现一个多Agent协作平台，让多个专门化的Agent协同工作，完成复杂任务。

### 应用场景示例

**场景：生成竞品分析报告**
```
用户输入: "帮我分析一下ChatGPT和Claude的差异，生成报告"

系统协作流程：
1. 规划Agent：分解任务（技术对比、功能对比、市场对比）
2. 研究Agent：搜索相关信息
3. 分析Agent：对比分析差异
4. 写作Agent：整合信息生成报告
5. 审查Agent：检查报告质量
```

---

## 核心需求

### 1. Agent抽象和设计
- 设计通用的Agent基类或接口
- 实现至少3种不同类型的Agent（如：研究、分析、写作）
- 每个Agent有明确的能力定义和输入输出规范

### 2. 工作流编排
- 定义工作流的描述方式（配置、代码或DSL）
- 实现工作流执行引擎
- 支持顺序执行、并行执行、条件分支

### 3. Agent间通信
- 设计Agent之间的消息传递机制
- 实现共享状态管理
- 处理Agent依赖关系

### 4. 错误处理
- Agent执行失败的重试机制
- 工作流的暂停、恢复、回滚
- 详细的执行日志

### 5. 应用场景（选择1-2个实现）
- **研究报告生成**：自动收集信息、分析、生成报告
- **代码审查**：多个Agent分别检查不同方面
- **客服系统**：Intent识别、信息查询、方案生成
- 或自己设计的场景

### 6. 设计思考（请在文档中说明）
- Agent能力边界划分的设计思路
- Agent协作中的依赖关系管理
- 多Agent协作效率的优化方向
- 系统调试与监控的实现方案
