# 方向A：AI/LLM应用

---

## 项目背景

在多轮对话场景中，用户与AI之间会产生大量上下文信息，但这些信息通常不会被长期利用。你需要设计并实现一个”记忆系统”，能够从多轮对话中自动总结结构化记忆，并在后续问答中进行检索与使用。

> **技术选型提示**：可以选择使用 RAG（向量检索）或 Graph（知识图谱）两种技术路线来实现记忆存储与检索功能。

### 应用场景示例

**场景：用户关系记忆与问答**

系统通过两阶段流程实现智能记忆与问答：

---

**阶段一：对话 → 记忆存储**

```
对话内容：
用户：今天我和兰姐出去玩了
助手：兰姐是谁呀
用户：兰姐是我老师，在黄明小学当语文老师，喜欢跳舞
助手：知道了

【触发流程一：存储记忆】

系统自动提取并存储结构化记忆：
{
  "name": "兰姐",
  "relation": "老师",
  "observation": "在黄明小学做语文老师，喜欢跳舞"
}
```

---

**阶段二：新对话 → 记忆检索**

```
（在新的对话会话中）

用户提问：
我可以去哪里找到兰姐？

【触发流程二：读取记忆】

系统检索记忆 → LLM生成回答：
"根据记忆，兰姐在黄明小学做语文老师，你可以去那里找她。"

引用的记忆来源：
- 兰姐 | 老师 | 在黄明小学做语文老师，喜欢跳舞
```

### 系统流程

#### 流程一：存储记忆流程（Memory Storage Pipeline）

将多轮对话转换为结构化记忆并存储：

**步骤：**

1. **对话输入**：接收多轮对话历史
2. **对话预处理**：提取用户发言，识别关键实体（人名、地点、关系等）
3. **LLM信息抽取**：通过提示词工程提取结构化信息（name / relation / observation），输出JSON格式
4. **记忆去重与更新**：
   - 检查相似记忆是否已存在
   - 如存在：合并更新
   - 如不存在：新增记忆
5. **记忆存储**：将记忆存储到向量数据库（RAG）或知识图谱（Graph），带元数据
6. **完成**：记忆存储成功


#### 流程二：读取记忆流程（Memory Retrieval Pipeline）

基于用户问题检索相关记忆并生成智能回答：

**步骤：**

1. **用户提问**：接收用户问题
2. **问题编码**：将问题转换为查询格式（向量嵌入或图查询）
3. **语义检索**：在向量数据库/知识图谱中检索相似记忆，返回Top-K相关记忆
4. **检查检索结果**：
   - **有相关记忆** → 进入步骤5
   - **无相关记忆** → LLM直接回答（无记忆上下文）
5. **上下文组装**：系统提示词 + 检索到的记忆 + 用户问题
6. **LLM生成回答**：基于记忆上下文生成回答，标注引用来源
7. **返回结果**：返回自然语言回答 + 引用的记忆（结构化）
8. **完成**



### 核心价值

- 从对话中自动沉淀长期记忆
- 支持结构化人物关系抽取
- 提供可追溯的记忆来源
- 提升多轮对话的智能性与连贯性

## 核心需求

### 1. 对话处理与记忆构建

- 支持多轮对话输入处理
- 从对话中提取关键信息（人物、关系、属性等）
- 设计记忆结构（如：name / relation / observation）
- 支持记忆的更新与去重

### 2. 相似度检索

- 选择合适的存储方案：向量数据库（RAG）或知识图谱（Graph）
- 设计记忆存储策略（结构化 + 文本）
- 实现基于语义的记忆检索
- 支持增量添加和更新记忆

### 3. 问答功能

- 实现基于记忆的检索流程（可选 RAG 或 Graph）
- 支持多轮对话上下文
- 能够结合记忆生成自然语言回答
- 处理无相关记忆的情况

### 4. API或界面

- 提供API接口或图形界面（二选一或都做）
- 支持对话输入、记忆查询、记忆管理
- 返回结构化结果（回答 + 引用记忆）

### 5. 设计思考（请在文档中说明）

- 记忆结构的设计思路
- 对话转记忆的抽取策略
- 检索准确度的优化方向
- 记忆规模增长下的性能与成本控制
